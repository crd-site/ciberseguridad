🛠️ Guía de instalación de GNS3 (modo local vs. servidor en VMware)

🧾 ¿Qué es GNS3?

GNS3 (Graphical Network Simulator-3) es un simulador de redes de código abierto que permite diseñar, configurar y probar redes complejas en un entorno virtual, sin necesidad de hardware físico.

Se utiliza ampliamente en el ámbito académico, profesional y de certificación (como Cisco, Fortinet, etc.), ya que permite emular routers, switches, firewalls y otros dispositivos, conectándolos como si se tratara de una red real.

GNS3 es ideal para aprender redes, practicar configuraciones, simular fallas o entrenar para exámenes de certificación, y se puede integrar con máquinas virtuales (como Kali Linux, Windows Server, pfSense, etc.) usando plataformas como VMware o VirtualBox.

Descarga: GNS3 y GNS3 VM

Documentación oficial: docs.gns3.com

❓¿Instalación local o con servidor en VMware?

  • Local: Todo corre en tu PC. Más simple, pero consume muchos recursos si usás varias máquinas virtuales o imágenes pesadas.
  • Con VMware: Usás una VM externa (GNS3 VM) que ejecuta los dispositivos, mientras tu PC actúa como cliente. Esto mejora el rendimiento, la estabilidad y facilita el uso de imágenes avanzadas (como Cisco IOU, ASA, etc.).

👉 Recomendado: Usar GNS3 VM en VMware para entornos de red más exigentes.

📘 Pasos para instalar GNS3 con VMware

  1. Descargar e importar la GNS3 VM
    – Descargá el archivo .ova desde la web oficial.
    – En VMware Workstation:
    File → Open
    y seleccioná el archivo.
    – Aceptá la configuración por defecto y finalizá la importación.
  2. Iniciar la GNS3 VM
    – Abrí la VM y esperá que arranque completamente.
    – La IP asignada aparecerá en pantalla (por ej. 192.168.56.101).
  3. Conectar GNS3 con la VM
    – Abrí GNS3 y, en el primer inicio, seleccioná “Run appliances on a remote server”.
    – Ingresá la IP de la VM y aceptá.
  4. Probar la conexión
    – Agregá un router o switch al proyecto.
    – Arrastrá un dispositivo y encendelo. Si funciona, ¡todo listo!
