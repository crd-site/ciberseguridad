🖥️ Guía – Primer proyecto y router en GNS3

📂 1. Abrir GNS3

Qué es: Es el primer paso para iniciar tu entorno de simulación y verificar que GNS3 esté listo para trabajar.

  1. Abrí GNS3 desde el acceso directo o menú de inicio.
  2. Revisá el estado de la GNS3 VM (esquina inferior derecha):
    🟢 Verde = VM conectada • 🔴 Rojo = VM apagada (encendé la VM en VMware/VirtualBox).

📁 2. Crear un nuevo proyecto

Qué es: Un “contenedor” donde se guarda toda la configuración, topología y archivos del laboratorio.

  1. Menú File → New Project (o Ctrl+N).
  2. Escribí un nombre descriptivo (ej.: Proyecto1_Router).
  3. Elegí la carpeta de destino y hacé clic en OK.

📡 3. Añadir el primer router

Qué es: El dispositivo principal para tu laboratorio. Puede ser IOSv (recomendado con GNS3 VM) o modelos emulados legacy (7200/3725) con Dynamips.

  1. En Devices, buscá Routers.
  2. Arrastrá un Cisco IOSv (o el que tengas) al área de trabajo.
  3. Si no aparece ninguno: Edit → Preferences → (Dynamips/IOU/VM) y Add para cargar la imagen/plantilla.

🔌 4. Conectar el router

Qué es: El cableado virtual que permite la comunicación entre dispositivos.

  1. Seleccioná el ícono Add a Link (cable).
  2. Elegí la interfaz del router (ej.: FastEthernet0/0 o GigabitEthernet0/0).
  3. Conectá a un switch genérico o a una nube/NAT para alcanzar otras redes o VMs.

▶️ 5. Encender el router

Qué es: Activar el dispositivo para iniciar su sistema operativo.

  1. Clic derecho sobre el router → Start.
  2. Esperá a que el ícono cambie a verde.

💻 6. Abrir consola y configuración básica

Qué es: La interfaz de línea de comandos (CLI) para administrar el router.

  1. Clic derecho → Console.
  2. Ingresá los comandos iniciales (ejemplo con Fa0/0):
enable                               
configure terminal                   
interface FastEthernet0/0            
ip address 192.168.1.1 255.255.255.0 
no shutdown                          
exit                                 
write memory

💡 Nota: Si no ejecutás write memory o copy running-config startup-config, la configuración se perderá al reiniciar.

🧪 7. Probar conectividad

Qué es: Confirmar que el router responde en red.

  1. Agregá un PC genérico (VPCS o QEMU/VM) a la misma red del router.
  2. Configurá la IP 192.168.1.2/24 y gateway 192.168.1.1 (si corresponde).
  3. Desde la consola del PC, probá:
ping 192.168.1.1

Si responde, ¡todo funciona! ✅

📜 8. Configuración de Access Lists (ACL)

🔎 ¿Qué es una ACL?

Una Lista de Control de Acceso (ACL) es un conjunto de reglas que filtran el tráfico de red en función de criterios como IP de origen/destino, protocolo o puerto. Se aplican en una interfaz (in/out) para controlar qué tráfico puede entrar o salir.

📚 Tipos de ACL y rangos

Tipo de ACL Rango clásico Rango extendido Qué filtra
Estándar 1–99 1300–1999 Solo IP de origen
Extendida 100–199 2000–2699 Origen, destino, protocolo y puerto

💡 Mnemotécnica: Estándar → números bajos → filtra menos. | Extendida → números altos → filtra más.

🧩 Estructura general

access-list <número> <permit|deny> <criterio> <wildcard mask>

✅ Ejemplo – ACL estándar

Escenario: Bloquear el acceso de una PC específica 192.168.10.50 a toda la red.

access-list 15 deny 192.168.10.50 0.0.0.0
access-list 15 permit any
interface FastEthernet0/0
ip access-group 15 in
  • Usá ACL estándar cuando solo necesites filtrar por IP de origen.
  • Menor complejidad y consumo que una extendida.

✅ Ejemplo – ACL extendida

Escenario: Permitir que la red 192.168.10.0/24 solo acceda al servidor web 172.16.1.100 (HTTP/80) y bloquear el resto.

access-list 110 permit tcp 192.168.10.0 0.0.0.255 host 172.16.1.100 eq 80
access-list 110 deny ip any any
interface FastEthernet0/0
ip access-group 110 in

Sobre deny ip any any: hay una deny implícita al final, pero dejarla explícita documenta mejor la política.

⚖️ Comparativa rápida

Característica ACL estándar ACL extendida
Filtro Solo IP de origen Origen + destino + protocolo + puerto
Complejidad Baja Alta
Uso típico Bloqueos generales Políticas específicas y detalladas
Eficiencia Mayor Menor
Ejemplo Bloquear host problemático Permitir solo HTTP hacia un servidor

🛠️ Troubleshooting – Problemas comunes

💥 Problema 🔍 Causa 🛠 Solución
Router no aparece No hay imágenes/plantillas cargadas Cargar imagen en Preferences → Dynamips/IOU/VM → Add
Router no enciende Imagen corrupta o RAM insuficiente Reimportar la imagen y asignar ≥ 256 MB
Consola no abre Cliente Telnet/Consola no configurado Configurar en Preferences → Console Applications
Sin conexión Interfaces apagadas o sin IP Usar show ip interface brief y no shutdown
Ping falla IP/máscara incorrecta en router/PC Verificar direcciones y máscara /24 en ambos extremos
